# Brownfield Decompose-Only Example

This example shows Uigwe starting directly at decomposition because an approved design already exists.

## Entry Mode

`decompose-only`

## Scenario

Extend the existing Uigwe documentation set with a future wrapper surface for invoking the protocol more directly, while preserving existing protocol and schema boundaries.

## Artifact Order

1. `design.packet.json`
2. `spec.md`
3. `rationale.md`
4. `plan.packet.json`
5. `goal-tree.json`

## What This Example Demonstrates

- how Uigwe can skip `deep-interview` and `brainstorming` when a strong design packet already exists
- why `brownfield` decomposition requires tighter file scope and stronger verification clarity
- how retained alternatives are still preserved even in `decompose-only` mode
